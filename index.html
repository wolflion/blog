title: github使用常见问题
date: 2015-10-23
categories: blog

1.github本地使用多个key

本地多个key

  • 不管linux还是win，要在.ssh/下新建个config。

2. remote origin already exists

fatal: remote origin already exists.

git remote rm origin
git remote add origin git@github.com:name/name.git

3. git push后提示 everything up-to-date git

everything up-to-date git

git status 查看状态
git log 查看日志
一般的情况是add或commit失败git commit

4. error:failed to push some refs to…….

执行git push origin master

error:failed to push som refs to…….

解决failed to push some refs to
git pull origin master//把远程服务器github上文件拉回来，再push上去。

5. The following untracked working tree files would be overwritten by checkout:

The following untracked working tree files would be overwritten by checkout: ReadMe.md

git clean -d -fx //不太熟悉
git stash //不太熟悉

命令详解

git init //当前目录下会创建一个.git的目录
git clone
git add
git commit -m 'comment'
git branch testbranch //创建分支
git checkout testbranch //切换到分支
git branch //查看分支
git branch -r //查看远程所有分支
git branch -a //查看本地和远程的所有分支
git push origin test//提交test分支到远程git库
git reset //不太熟悉
git rebase //不太熟悉

gitconfig配置文件

win环境下在C:\Documents and Settings\$USER下。
